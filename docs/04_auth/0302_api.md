# 0302 ログイン - API

## エンドポイント

| メソッド | パス | 認証 | 説明 |
|----------|------|------|------|
| POST | `/api/login` | 不要 | ログイン |
| POST | `/api/logout` | 必要 | ログアウト |
| GET | `/api/me` | 必要 | 認証ユーザー情報取得 |

---

## 1. Controller の作成

```bash
sail artisan make:controller Api/AuthController
```

`app/Http/Controllers/Api/AuthController.php` を実装してください。

```php
public function login(LoginRequest $request): JsonResponse
{
    if (!Auth::attempt($request->only('email', 'password'))) {
        return response()->json(
            ['message' => 'メールアドレスまたはパスワードが正しくありません。'],
            401
        );
    }

    $request->session()->regenerate();

    return response()->json(['message' => 'ログインしました。']);
}

public function logout(Request $request): JsonResponse
{
    Auth::logout();
    $request->session()->invalidate();
    $request->session()->regenerateToken();

    return response()->json(['message' => 'ログアウトしました。']);
}

public function me(): JsonResponse
{
    return response()->json(Auth::user());
}
```

---

## 2. ルーティングの設定

`routes/api.php` に追加してください。

```php
Route::post('/login', [AuthController::class, 'login']);

Route::middleware('auth:sanctum')->group(function () {
    Route::post('/logout', [AuthController::class, 'logout']);
    Route::get('/me', [AuthController::class, 'me']);
});
```

## リクエスト / レスポンス

**ログイン成功時 200**

```json
{ "message": "ログインしました。" }
```

**ログイン失敗時 401**

```json
{ "message": "メールアドレスまたはパスワードが正しくありません。" }
```

## 学習

### 今何をしたか

管理者がログイン・ログアウトするための API を実装しました。

| 用語 | 説明 |
|------|------|
| 認証 | ユーザーが誰かを確認する処理 |
| セッション | ログイン状態をサーバーに保持する仕組み |
| `Auth::attempt()` | メールアドレスとパスワードを照合する Laravel の関数 |
| `session()->regenerate()` | セッション固定攻撃を防ぐためにセッション ID を再生成する |
| 401 Unauthorized | 認証が必要、または認証に失敗した際に返す HTTP ステータスコード |
