# 0205 Sail 起動・初期設定

## 1. エイリアスの設定（任意）

毎回 `./vendor/bin/sail` と打つのが大変なため、短縮エイリアスを設定しておきます。

`~/.zshrc` に以下を追記してください。

```bash
alias sail='./vendor/bin/sail'
```

反映します。

```bash
source ~/.zshrc
```

以降は `sail` と打つだけで動きます。

---

## 2. Sail の起動

```bash
./vendor/bin/sail up -d
```

エイリアスを設定した場合は以下でも OK です。

```bash
sail up -d
```

初回は Docker イメージのダウンロードが発生するため、数分かかります。

---

## 3. アプリケーションキーの生成

```bash
sail artisan key:generate
```

---

## 4. マイグレーションの実行

```bash
sail artisan migrate
```

---

## 5. 動作確認

ブラウザで [http://localhost](http://localhost) を開き、Laravel のウェルカム画面が表示されれば OK です。

---

## Sail の停止

```bash
sail down
```

---

## チェックリスト

- [ ] `sail up -d` が成功している

## 学習

### 今何をしたか

Docker コンテナを起動し、Laravel アプリケーションが動く状態にしました。

| 用語 | 説明 |
|------|------|
| `sail up -d` | Docker コンテナをバックグラウンドで起動する |
| `sail artisan` | コンテナ内で `php artisan` コマンドを実行する |
| `key:generate` | アプリケーションの暗号化に使うキーを生成する |
| マイグレーション | データベースのテーブル構造をコードで管理し、適用する仕組み |
