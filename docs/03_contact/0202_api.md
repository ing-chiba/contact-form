# 0202 お問い合わせ - API

## エンドポイント

| メソッド | パス | 説明 |
|----------|------|------|
| POST | `/api/contacts` | お問い合わせ送信 |

---

## 1. Model の作成

```bash
sail artisan make:model Contact
```

---

## 2. Controller の作成

```bash
sail artisan make:controller Api/ContactController
```

`app/Http/Controllers/Api/ContactController.php` を実装してください。

```php
public function store(StoreContactRequest $request): JsonResponse
{
    Contact::create($request->validated());

    return response()->json(['message' => 'お問い合わせを受け付けました。'], 201);
}
```

---

## 3. ルーティングの設定

`routes/api.php` に追加してください。

```php
Route::post('/contacts', [ContactController::class, 'store']);
```

---

## リクエスト / レスポンス

**リクエスト**

```json
{
    "name": "山田 太郎",
    "company": "株式会社サンプル",
    "email": "yamada@example.com",
    "message": "お問い合わせ内容"
}
```

**成功時 201**

```json
{
    "message": "お問い合わせを受け付けました。"
}
```

**バリデーションエラー時 422**

```json
{
    "message": "The given data was invalid.",
    "errors": {
        "email": ["メールアドレスの形式が正しくありません。"]
    }
}
```

## 学習

### 今何をしたか

お問い合わせを受け取る API エンドポイントを作成しました。

| 用語 | 説明 |
|------|------|
| REST API | HTTP メソッド（GET / POST など）でリソースを操作する設計スタイル |
| Model | DB のテーブルに対応する PHP クラス。データの取得・保存を行う |
| Controller | リクエストを受け取り、処理を行い、レスポンスを返すクラス |
| ルーティング | URL と Controller のメソッドを対応付ける設定 |
| 201 Created | リソースの作成に成功したときに返す HTTP ステータスコード |
