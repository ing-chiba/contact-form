# 0401 管理者 - API

## エンドポイント

| メソッド | パス | 認証 | 説明 |
|----------|------|------|------|
| GET | `/api/admin/contacts` | 必要 | お問い合わせ一覧取得 |

---

## 1. Controller の作成

```bash
sail artisan make:controller Api/Admin/ContactController
```

`app/Http/Controllers/Api/Admin/ContactController.php` を実装してください。

```php
public function index(): JsonResponse
{
    $contacts = Contact::orderBy('created_at', 'desc')->get();

    return response()->json(['data' => $contacts]);
}
```

---

## 2. ルーティングの設定

`routes/api.php` に追加してください。認証ミドルウェアで保護します。

```php
Route::middleware('auth:sanctum')->prefix('admin')->group(function () {
    Route::get('/contacts', [Admin\ContactController::class, 'index']);
});
```

---

## レスポンス

**成功時 200**

```json
{
    "data": [
        {
            "id": 1,
            "name": "山田 太郎",
            "company": "株式会社サンプル",
            "email": "yamada@example.com",
            "message": "お問い合わせ内容",
            "created_at": "2024-01-01T00:00:00.000000Z"
        }
    ]

```

**未認証時 401**

```json
{ "message": "Unauthenticated." }
```

## 学習

### 今何をしたか

管理者専用のお問い合わせ一覧 API を作成しました。
ミドルウェアを使って認証済みユーザーのみアクセスできるよう保護しています。

| 用語 | 説明 |
|------|------|
| ミドルウェア | リクエストが Controller に届く前後に処理を挟む仕組み |
| `auth:sanctum` | Sanctum による認証チェックを行うミドルウェア |
| `prefix` | ルートグループに共通のパスプレフィックスを付ける設定 |
